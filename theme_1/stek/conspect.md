# Конспект по алгоритмам на стеке

## Определение стека

Стек — структура данных, работающая по принципу LIFO (Last In — First Out). Последний добавленный элемент извлекается первым. Основные операции: добавление элемента на вершину (push), удаление элемента с вершины (pop), просмотр верхнего элемента без удаления (peek).

Стек применяется в задачах, где требуется временное хранение данных с последующим извлечением в обратном порядке, обработка вложенных структур, реализация отложенных операций.

## Монотонный стек

Монотонный стек — это стек, в котором элементы всегда упорядочены (строго или нестрого) по возрастанию или убыванию. При добавлении нового элемента из вершины удаляются все элементы, нарушающие порядок.

Применяется для задач, где нужно найти ближайший больший/меньший элемент, поддерживать последовательность с ограничениями на порядок, проверять возможность сортировки через временное хранилище.

## Задача 1. Постфиксная запись

### Условие
Выражение в постфиксной записи (обратной польской записи) содержит цифры и операции +, -, *, разделённые пробелами. Операция записывается после двух операндов. Требуется вычислить значение выражения.

### Принцип решения
Стек хранит операнды. При чтении числа оно помещается в стек. При чтении операции из стека извлекаются два последних операнда (первый извлечённый — правый операнд, второй — левый), выполняется операция, результат помещается обратно в стек. После обработки всех токенов в стеке остаётся единственное значение — результат.

### Код
```python
expression = input().strip().split()

stack = []

for token in expression:
    if token in '+-*':
        b = stack.pop()
        a = stack.pop()
        if token == '+':
            stack.append(a + b)
        elif token == '-':
            stack.append(a - b)
        elif token == '*':
            stack.append(a * b)
    else:
        stack.append(int(token))

print(stack[0])
```

### Разбор
Алгоритм последовательно читает токены. Числа накапливаются в стеке. Когда встречается оператор, два верхних числа извлекаются, результат вычисления возвращается в стек. Так как операторы бинарные, стек всегда содержит достаточное количество элементов. Постфиксная форма гарантирует корректный порядок извлечения операндов.

## Задача 2. Сортировка вагонов

### Условие
Поезд из N вагонов подходит к тупику. Вагоны пронумерованы от 1 до N. С пути 1 можно завезти в тупик K вагонов (операция 1), из тупика на путь 2 — вывезти K вагонов (операция 2). Каждый вагон заезжает в тупик один раз и выезжает один раз. Требуется получить порядок 1, 2, ..., N на пути 2 или определить невозможность.

### Принцип решения
Тупик является стеком. Ожидаемый следующий вагон — current, начинается с 1. Поезд на пути 1 обрабатывается последовательно. Если первый вагон из оставшихся на пути 1 равен ожидаемому, он сразу отправляется на путь 2 (операции 1 и 2 по одному вагону). Если верхний вагон в тупике равен ожидаемому, он вывозится на путь 2 (операция 2). Иначе вагон с пути 1 завозится в тупик (операция 1). Если вагонов на пути 1 больше нет, а ожидаемый вагон не находится на вершине стека, сортировка невозможна.

### Код
```python
n = int(input())
wagons = list(map(int, input().split()))

stack = []
actions = []
current = 1
idx = 0

while current <= n:
    if idx < n and wagons[idx] == current:
        actions.append((1, 1))
        actions.append((2, 1))
        idx += 1
        current += 1
    elif stack and stack[-1] == current:
        actions.append((2, 1))
        stack.pop()
        current += 1
    elif idx < n:
        actions.append((1, 1))
        stack.append(wagons[idx])
        idx += 1
    else:
        break

if current == n + 1 and not stack:
    for action in actions:
        print(action[0], action[1])
else:
    print(0)
```

### Разбор
Алгоритм имитирует пошаговое управление составом. Переменная current хранит номер вагона, который должен следующим покинуть тупик на путь 2. Переменная idx указывает на текущий обрабатываемый вагон на пути 1. Ветвление определяет возможное действие: прямой проход, вывоз из тупика, завоз в тупик. Если после обработки всех вагонов стек пуст и все вагоны выданы в правильном порядке, последовательность действий выводится. В противном случае выводится 0.

## Задача 3. Конвейер

### Условие
Контейнеры поступают на конвейер из цеха А. Накопитель позволяет на каждом шаге: переместить первый контейнер с ленты в цех В, переместить первый контейнер с ленты в склад (склад работает как стек), переместить верхний контейнер со склада в цех В. Требуется определить, можно ли отсортировать контейнеры по степени срочности (меньшее число — выше срочность).

### Принцип решения
Склад является стеком. Ожидаемая последовательность — отсортированный по возрастанию массив степеней срочности. Алгоритм проходит по входной последовательности, каждый контейнер сначала помещается в стек. Затем, пока вершина стека совпадает с очередным ожидаемым значением, контейнер извлекается из стека и счётчик ожидаемых увеличивается. Если в конце все контейнеры извлечены в правильном порядке, сортировка возможна.

### Код
```python
def can_order(k, priorities):
    stack = []
    expected = sorted(priorities)
    idx = 0
    
    for priority in priorities:
        stack.append(priority)
        while stack and stack[-1] == expected[idx]:
            stack.pop()
            idx += 1
    
    return 1 if idx == k else 0

n = int(input())
results = []

for i in range(n):
    data = input().strip().split()
    k = int(data[0])
    priorities = [float(x) for x in data[1:1 + k]]
    results.append(can_order(k, priorities))

print("\n".join(map(str, results)))
```

### Разбор
Функция can_order моделирует работу накопителя. Сортированный список expected задаёт целевой порядок. При проходе по входной последовательности каждый элемент сначала помещается в стек. Цикл while извлекает из стека все элементы, которые совпадают с текущим ожидаемым, и увеличивает idx. Если после обработки всех контейнеров idx достиг K, значит, все контейнеры удалось выдать в правильном порядке. Проверка использует свойство стека: извлечение возможно только с вершины, поэтому порядок извлечения определяется последовательностью помещений и совпадений с ожидаемыми значениями.

## Сравнение подходов

В первой задаче стек выполняет роль накопителя операндов для вычислений. Во второй задаче стек моделирует железнодорожный тупик с ограничением на порядок выезда. В третьей задаче стек выступает в роли склада-накопителя, а алгоритм использует принцип монотонного извлечения по совпадению с отсортированной последовательностью.

Все три задачи демонстрируют ключевое свойство стека: последний помещённый элемент становится доступным первым. Это свойство определяет возможность или невозможность получить требуемый порядок на выходе.
